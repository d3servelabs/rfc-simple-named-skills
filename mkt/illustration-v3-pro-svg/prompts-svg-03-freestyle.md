# 03 — REINVENTING THE ADDRESS WHEN THE BUILDING ALREADY HAS ONE

Create an animated SVG infographic contrasting three new identity systems vs the one that already exists.

On one side: three teams each building a brand-new identity system from scratch — a skill marketplace with invented namespaces (confusing lookalikes like "ns:acme-official" vs "ns:acme_real"), a GitHub org/repo hierarchy as identity, and an Ethereum smart contract registry with hex addresses and gas fees. All to answer: "Is this really from who it claims to be?"

On the other side: a simple browser address bar showing "https://acme.com" with a padlock. Already on every business card, email signature, and invoice. Already verified by DNS, TLS, DNSSEC, and Certificate Transparency. 4.5 billion users. Cost to use: $0.

The domain system already solved this problem. Why are we reinventing the address?
